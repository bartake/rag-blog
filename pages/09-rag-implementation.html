Building a Simple RAG App

The minimal flow: Ingest documents (parse, chunk) → Store embeddings in a vector DB → Query: embed user question, retrieve top-K chunks, inject into prompt, call LLM, return answer.

Ingest Docs

Load PDFs, markdown, or text. Split into chunks (e.g. 500 tokens with overlap). Attach metadata (source, page, section).

Store Embeddings

Use an embedding API (OpenAI, Cohere, etc.) or local model. Upsert chunks + vectors into Pinecone, Chroma, pgvector, or similar.

Query + Generate

Embed the question, run similarity search, build prompt with top chunks, call LLM, stream or return the response.

Frameworks

LangChain: composable chains, loaders, vector stores. Good for prototyping. LlamaIndex: focused on indexing and retrieval; strong for complex ingestion and query graphs. Both support multiple backends.

APIs

OpenAI: text-embedding-3-* for embeddings, GPT-4/3.5 for generation. Simple REST API. Anthropic, Cohere, and others offer similar embeddings + LLM combos. Consider cost, latency, and region.
