15. Cost Optimization

Embedding cost, storage, retrieval vs LLM cost, caching

Embedding Cost Strategies

Batch embedding calls instead of per-document. Use cheaper models for indexing vs query-time if quality allows. Cache embeddings; avoid re-embedding unchanged docs. Consider open-source/local models to eliminate API cost.

Storage Optimization (Quantization)

Quantize vectors (float32 → int8) to cut storage and often improve speed. PQ (Product Quantization) and scalar quantization trade slight recall for big savings. Test recall impact before deploying.

Retrieval Cost vs LLM Cost

LLM calls usually dominate. Reduce context length when possible; use smaller models; cache responses. Retrieval cost scales with index size and QPS; optimize with indexing and caching.

Caching Embeddings and Responses

Cache query embeddings for identical/similar queries. Cache full LLM responses when query+context match. Implement TTL and invalidation on doc updates.
