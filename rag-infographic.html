Retrieval-Augmented Generation (RAG)

Grounding LLMs in your data — without fine-tuning

RAG augments large language models with external knowledge at inference time. Instead of relying solely on pre-trained weights (which have cutoff dates and can hallucinate), RAG retrieves relevant documents from a knowledge base, injects them into the prompt, and lets the LLM generate answers grounded in that context. This enables up-to-date, factual, and citeable responses from private or proprietary data—without retraining the model.

RAG Workflows
Ingestion Parse, chunk, embed, store
→
Retrieval Embed query, find top-K chunks
→
Generation Inject context, call LLM, return answer
End-to-end: Documents → chunks → embeddings → vector DB. At query: user question → embedding → similarity search → retrieved chunks + question → LLM → answer.
Embeddings & Storage Techniques — Use Cases
Dense Embeddings (Single Vector)
One vector per chunk. Semantic similarity via cosine/dot product. Fast, scalable.
Use: General QA, documentation search, FAQ bots
Parent-Child Chunking
Small children for retrieval, return larger parent for context. Granular search + coherent context.
Use: Long documents, technical manuals, legal docs
Hybrid (Vector + BM25)
Combine semantic and keyword search. RRF fusion. Covers exact terms + paraphrases.
Use: Product search, support tickets, names/IDs
Metadata Filtering
Filter by source, date, tenant, language. Pre/post filter on vector search.
Use: Multi-tenant SaaS, time-sensitive docs, localized content
Re-ranking (Cross-encoder)
Retrieve many, re-rank top-20→top-5. Query+passage scored together. Higher accuracy.
Use: High-stakes answers, legal/medical, when precision > speed
Context Compression
LLM summarizes long chunks before injection. Fewer tokens, key info preserved.
Use: Large context limits, cost-sensitive, summarization-focused

Key Concepts
Why RAG? (Hallucination)
LLMs make up plausible but false answers. RAG grounds responses in retrieved docs—reduce fabrication.
RAG vs Fine-Tuning
RAG: update knowledge instantly. Fine-tune: change behavior/style. Use both for hybrid (RAG + fine-tuned model).
Production Checklist
  • Chunking: Semantic or overlap; tune size for your domain
  • Embeddings: Match ingest + query model; cache when possible
  • Retrieval: Top-K + optional re-rank; consider hybrid
  • Prompts: Clear instruction to "answer only from context"; say "I don't know" when unsure
  • Observability: Log queries, retrieval results, token usage, latency
